정처기 이론
스니핑 : (보안)네트워크에서 남의 패킷을 도청하는 것
즉각 갱신 기법 : (회복기법)트랜잭션이 데이터를 변경함녀 트랜잭션이 부분 완료되기 전이라도 즉시 실제 DB에 그 내용을 반영하는 기법
디자인 패턴의 3가지 종류 : 행위, 생성, 구조
DAC(임의 접근 통제 Discretionary Access Control) : 데이터에 접근하는 사용자의 신원에 따라 접근 권한을 부여하여 제어하는 방식

데이터 모델 구성요소
    연산 : 데이터베이스에 저장된 실제 데이터를 처리하는 작업에 대한 명세
    구조 : 논리적으로 표현된 객체 타입들 간의 관계로서 데이터의 구성 및 정적 성질을 표현

TCP/IP 기반의 인터넷 통신 서비스에서 ip와 조합해서 통신 중에 발생하는 오류 처리와
전송 경로 변경등을 위한 무연결 전송 프로토콜 -> ICMP (osi 네트워크 계층에 속함)

릴레이션 A,B가 있을 때 렐레이션 B의 족ㄴ에 맞는 것들만 릴레이션 A에서 분리해서 프로젝션하는 연산
-> 기호 / (나눗셈)

헝가리안 표기법 : 변수명 작성시 변수의 자료형을 알 수 있도록 의미를 포함해서 변수명을 작성하는 것

리팩토링 : 프로그램 쉽게 이해하고 수정하여 빠르게 개발할 수 있도록 하기 위함이다.

형상 통제 : 식별된 형상 항목에 대한 변경 요구를 검토해서 현재의 기준선이 잘 반영될 수 있도록 조정하는 작업에

OSFP : RIP의 단점을 해결하여 새로운 기능을 지원하는 인터넷 프로토콜
최단 경로 탐색에 다익스트라 알고리즘 사용, 라우팅 정보에 변화가 생길 경우 변화도니 정보만 네트워크 내의 모든 라우터에서 알림
,인터넷 망에서 이용자가 최단 경로를 선정할 수 있도록 라우팅 정보에 노드 간의 거리 정보, 링크 상태 정보를 실시간으로 해서 
최단 경로로 라우팅을 지원한다.

IPS(IPSec) : 네트워크 트래픽에 대해 IP 계층에서 
IP 패킷 단위의 데이터 변조 방지 및 은닉 기능을 제공하는 네트워 계층에서의 보안 통신 규격

Observer 패턴 : 한 객체의 상태가 바뀌면 그 객체에 의존하는 다른 객체들에게 자동으로 연락이 가서 자동으로
내용이 갱신되는 패턴

SOAP : HTTP, HTTPS ,SMTP등을 사용하여 xml 기반의 메시지를 네트워크 상에서 교환하는 프로토콜
